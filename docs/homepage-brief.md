# Homepage Brief

## 1. Business Goal
- 기존 순차형 홈페이지 리뉴얼 프로세스를 AI 통합 파이프라인으로 대체한다.
- 내부 팀이 반복적으로 사용할 수 있는 운영 방식과 명령 세트를 만든다.
- KPI는 제작 리드타임 단축, 산출물 왕복 횟수 감소, 프리뷰 도달 시간 단축으로 둔다.

## 2. Audience
- Primary audience: 그로스센터 내부 실무자, PM, 디자이너, 프론트엔드 개발자
- Secondary audience: 다른 팀의 홈페이지 리뉴얼 담당자
- Current pain points: IA 작성, 디자인 컨펌, 프론트 구현이 순차적으로 흘러 병목이 생김

## 3. Offer
- Product or service: Claude Code와 Figma MCP를 활용한 홈페이지 리뉴얼 파이프라인
- Core value proposition: 한 명 또는 소수 인원이 기획, 디자인, 구현 초안을 빠르게 연결할 수 있다.
- Main proof points: 텍스트 기반 IA 생성, Figma handoff 자동화, 구현 스펙 및 코드 작업 연결

## 4. Conversion
- Primary CTA: 워크플로우 도입 문의 또는 파일럿 프로젝트 시작
- Secondary CTA: 데모 요청 또는 내부 세션 요청
- Expected user journey after click: 소개 확인 후 도입 상담 또는 파일럿 범위 협의

## 5. Brand Direction
- Tone: 실무적이고 빠르며 과장 없는 방식
- Visual constraints: 기존 에이전시 스타일보다 운영 체계와 속도를 강조
- Reference sites: 추후 추가 필요

## 6. Content Inputs
- Existing copy available: NOTE.md의 문제 정의와 제안 흐름
- Existing assets available: 없음
- Missing assets: 로고, 실제 사례, 성과 수치, 시각 레퍼런스

## 7. Technical Constraints
- Preferred stack: Next.js 우선 검토
- Existing CMS or API: 미정
- Analytics tools: 미정
- Deployment target: Vercel 또는 Netlify 검토

## 8. Experiment Scope
- Hypothesis 1: 기존 제작 방식 대비 첫 프리뷰 도달 시간이 크게 줄어든다.
- Hypothesis 2: 단일 담당자가 생성한 초안이라도 팀 단위 협업 시작점으로 충분하다.
- Segments to test: 내부 실무팀, 파일럿 대상 팀

## 9. Delivery Constraints
- Deadline: 미정
- Review owner: 미정
- Must-have sections: 문제 정의, 제안 워크플로우, 단계별 산출물, 도입 방식, CTA
- Nice-to-have sections: 사례, FAQ, 비교표, 실험 설계 예시

## 10. Assumptions
- 현재 목표는 실제 퍼블릭 홈페이지보다 워크플로우 설명용 랜딩 페이지에 가깝다.
- 디자인 시스템과 실제 브랜드 자산은 아직 없는 상태다.
- 우선순위는 자동화 가능한 작업 흐름을 정리하는 것이다.
