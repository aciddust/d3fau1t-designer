## 내가 생각한 것

기존의 [기획 → 디자인 → 개발] 순차 구조를 [프롬프트/데이터 → AI 통합 빌드 → 피드백 루프] 구조로 전환

| 단계 | 핵심 도구 (Stack) | 액션 (Action) |
| - | - | - |
| 1. 구조설계 | Calude Sonnet | IA와 유저 플로우를 텍스트/Mermaid 차트로 즉시 생성 |
| 2. 디자인 자동화 | Figma MCP | Claude 내에서 직접 Figma 캔버스에 레이아웃 및 스타일 가이드 생성 |
| 3. 코드 구현 | Claude Code (Plugin: Command) | 디자인 사양을 읽어 즉시 프론트엔드(React/Next.js/SvelteKit) 코드로 변환 |
| 4. 배포 및 검증 | Vercel/Netlify | 코드 푸시와 동시에 프리뷰 링크 생성 및 그로스 실험(A/B Test) 연결 |
